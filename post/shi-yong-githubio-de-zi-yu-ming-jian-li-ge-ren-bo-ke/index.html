使用 GitHubIo 的子域名建立个人博客

2025-03-18 2 min read # Cats_and_Dogs

首先

GitHub pages 分为个人主页、组织主页（organization）和项目主页，前二者的域名是最简便直接的 <user-name>.github.io、<organization-name>.github.io，显然此类域名与个人或组织的 github 账号一一对应，如果直接拿来作为博客主页似乎有点浪费......（这么简便的网址更应该拿来索引一个精彩的个人简介页面之类的🤪）。项目主页没有个数限制，且项目主页的地址也只比前两者多了一级索引而已，因此我们完全可以用它来建立个人博客网站。

然后

项目主页的建立流程和个人主页完全相同，只需注意两点：

  1. 新建仓库时，将仓库命名为 projectname.github.io，不要和个人主页或组织主页重名。以我的博客项目为例，仓库名为 BouseBlog.github.io
  2. 要注意将配置文件 _config.yml 中的 baseurl 设置为你的项目名，以我的博客项目为例，设置为 baseurl: BouseBlog.github.io/。
    1. 配置文件是 _config.yml，因为我默认读者都是按照 GitHub 官网给的教程使用 Jekyll 创建静态页面。
    2. 若不设置，在此项目仓库中发布的网页对同仓库其他页面的索引将默认从个人主页开始，通过设置 baseurl 将默认的 <user-name>.github.io/ 修改为 <user-name>.github.io/projectname.github.io/），这就是上面说的 “多了一级索引”。
