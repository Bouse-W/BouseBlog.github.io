配置 MiniKube | KIND 难蚌实录

2026-05-02 2 min read

系统配置

Intel x86-64，Ubuntu18.04（清朝老兵说是），Docker24.0.2

难蚌实录

最初选用 MiniKube，--driver=docker

组件太小，系统太老（

从各种报错信息中怀疑了 iptables 版本太低（v1.6.1）、AppArmor 拦截太严格、运行内存不足、cgroup 驱动冲突（宿主机和 Docker 都使用 systemd ）等原因

最终没能得逞，遂放弃

现在想来应该就是宿主机 iptables 版本过低，和 kubernetes 镜像中的 iptables 不匹配的原因。之后换用 kind 遭遇了同样的问题

通过 go install sigs.k8s.io/kind@latest 安装了最新版 kind，执行 kind create cluster 默认拉取最新的 kubernetes 镜像来构建集群

构建集群时 kind 会执行一系列 docker exec 操作，调用镜像中自带的环境配置脚本

环境配置将涉及：向宿主机的 iptables 写入一些特定规则，再从宿主机导出 netfilter 规则表，再导入镜像，以确保镜像内的 DNS 请求能被正确转发到宿主机的 IP。当镜像和宿主机的 iptables 版本不一致时，一些 netfilter 规则互不兼容，导出的规则表将用 Unsupported 填充不兼容的条目，导致镜像无法正确解析规则表，最终配置环境失败

PS

kind 自动拉取镜像大概率是失败的，因为 Docker Hub 很早就被 BAN 了。可以根据 kind 报错信息从国内备份站点手动拉取，或者在 daemon.json 配置使用国内镜像站

解决办法

合适的就很好

在 Intel x86-64，Ubuntu18.04，Docker24.0.2 的系统上使用 kind@latest，kindest/node v1.26.6 即可

构建集群命令：

kind create cluster --image=kindest/node v1.26.6
