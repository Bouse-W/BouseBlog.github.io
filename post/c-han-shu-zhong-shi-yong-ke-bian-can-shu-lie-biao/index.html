我的实验代码
#include <stdio.h>
#include <stdarg.h>

void test_change_param (char *, int, ...);

int main()
{
    char * test_string = "test changing params";
    unsigned int test_int = 98765;

    test_change_param (test_string, test_int);

    return 0;
}

void test_change_param (char * args, ...)
{
    va_list param_list;         // 定义参数列表变量

    va_start (param_list, args);    // 将列表变量 param_list 指向参数列表的第一个元素 "args" 

    // va_arg (ap, type) 获取可变参数列表中的下一个参数。
    // ap 是一个 va_list 类型的变量，type 是下一个参数的类型。该宏返回参数列表中类型为 type 的参数的值，并将 ap 指向下一个参数。
    printf ("the first unsined int parameter = %d", va_arg (param_list, unsigned int));

    // 清理为 valist 保留的内存，避免内存泄露
    va_end(param_list);

    return;
}
运行结果
一点说明
  • 含有可变参数的函数，在声明、定义时，"..."（省略号）出现即代表着：和它紧邻的前一个参数会是可变参数列表的第一个元素，在函数体中使用 v_start () 初始化列表变量时，必须将此紧邻参数作为列表变量的第一个元素，如若不然，编译即告警 [Warning] second parameter of 'va_start' not last named argument [-Wvarargs]
  • 使用可变参数函数大法，可以显著提高函数定义的灵活性，丰富函数功能，非常的新鲜，非常的美味
  • 有可变参数，自然有可变参数宏

大义觉迷

特别注意

实验代码中的写法是风险极高、危害极大、非常幼稚的😒，它会无可避免地造成内存访问越界的漏洞。

示例

test_change_param (test_string, test_int);

...

void test_change_param (char * args, ...)
{
    va_list param_list;         // 定义参数列表变量

    va_start (param_list, args);    // 将列表变量 "param_list" 指向参数列表的第一个元素 "args" 

    printf ("the first unsined int parameter = %d", va_arg (param_list, unsigned int));
    // 从这里开始越界访问。可变参数列表里只有一个元素，但还能继续向后访问
    printf ("the second unsigned int parameter = %d", va_arg (param_list, unsigned int));
    printf ("the third unsigned int parameter = %d", va_arg (param_list, unsigned int));

    va_end(param_list);

    return;
}

以上示例代码中，编译、执行都不会报错，但事实上对内存进行了越界访问。内存很容易就被看光了（虽然不会被改掉）

怎么办呢？

#include <stdio.h>
#include <stdarg.h>

void test_change_param (char *, int, ...);          // 函数声明和定义时，约定俗成地显式传递可变参数的个数

int main()
{
    char * test_string = "test changing params";
    unsigned int test_int = 98765;

    test_change_param (test_string, 1, test_int);

    return 0;
}

void test_change_param (char * args, int argv, ...)
{
    va_list param_list;

    va_start (param_list, argv);

    if (0 != argv)          // 在访问可变参数列表时，进行每一跳前首先判断是否已到达末尾
    {
        printf ("the first unsined int parameter = %d", va_arg (param_list, unsigned int));
        argv--;             // 每一跳进行后，立刻使未访问参数个数自减
    }

    // 清理为 valist 保留的内存，避免内存泄露
    va_end(param_list);

    return;
}

只能对变参函数的用法做出一些约束，在上述示例可以简要归为 4 点

  • 函数声明和定义时，约定俗成地显式传递可变参数的个数，作为最后一个固定参数。
  • 在一次调用中，传递可变参数时，尽量，尽量，尽量要求其类型相同。
    • 若可变参数列表中存在多种不同类型的元素，则在使用 va_start() 访问时每一跳的步进值可能不为 1，先前传递的可变参数个数就用不上了，该越界还是会越界。
    • 比如前两个是 int，第三个是 char，然后第一步就去找 char
    • 我说的是尽量相同，因为要迎合 va_start ()。艺高人胆大的同学，或者胆大的同学，可以抛弃 va_start()，手动操纵指针访问目标元素 ，也就不太需要在意这条约束了。
  • 在访问可变参数列表时，进行每一跳前首先判断是否已到达末尾
  • 每一跳进行后，立刻使未访问参数个数自减
  • 可变参数列表使用完毕，立刻清理为列表变量保留的内存，避免内存泄露
