NVIC 与 EXTI 的区别与联系

2025-03-18 3 min read # Embeded_programming

GPT 3.5

  • 当涉及到 STM32F103 固件库中的 NVIC（Nested Vectored Interrupt Controller）和 EXTI（External Interrupt）时，它们有以下区别

    • 功能和作用
      • NVIC 是一个用于处理多种中断的硬件模块，它负责管理中断优先级和中断嵌套（Nested）。
      • EXTI 用于处理外部中断，它负责监测和处理来自外部引脚的电平或边沿变化触发的中断。
    • 地位和结构
      • NVIC 属于 STM32F103 系列微控制器的核心内核组件之一。它是中央处理单元（CPU）内的一部分，用于管理和控制中断。NVIC 具有优先级级别和向量表，并支持嵌套中断的处理。
      • EXTI 是 STM32F103 固件库中的外围组件之一。它是专门用于处理特定外部引脚上的中断触发事件的模块。
    • 中断管理
      • NVIC管理器负责中断的优先级分配和中断处理的嵌套。通过配置NVIC寄存器，可以控制哪些中断可嵌套、中断的优先级以及中断向量表的位置等
      • EXTI用于接收和处理外部中断。它监视特定引脚上的电平或边沿变化，并触发相应的中断请求，然后由NVIC将其分配给适当的中断处理程序。
    • 配置和使用
      • 在使用 NVIC 之前，首先需要根据应用程序的要求，对中断优先级进行配置和初始化。这包括设置中断向量表、中断优先级和使能中断的相关寄存器等。
      • EXTI需要配置外部引脚以及相关的触发方式（电平触发或边沿触发）。当外部引脚发生变化时，会触发相应的中断请求，并由NVIC控制相应中断的处理。

  • 总结：NVIC负责控制和管理中断的优先级和嵌套，而EXTI用于处理特定外部引脚上的中断触发事件。它们在STM32F103固件库中具有不同的功能和作用，并一起协同工作以实现中断处理的功能。

NVIC 是 Cortex-M3 内核处理器微架构的一部分，Cortex-M3 基于 ARMv7-M 指令集，由 ARM 公司设计。

EXTI 是 STM32 系列 MCU 的一个片上外设，由芯片公司设计制造。

Cortex-M3 微架构示意图，红色线框中是 NVIC

STM32F 系列 MCU 总线架构图，红色线框中是 Cortex-M3 内核处理器，注意蓝色线框中的 EXTI

建议搞懂微架构和指令集的区别，弄明白 Cortex-M3 和 ARMv7-M 的关系。

EXTI 作为一个很好用的外设，具体的使用方法可以去参考官方的手册。
