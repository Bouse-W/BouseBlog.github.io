The beginning of all

2025-03-18 2 min read # Cats_and_Dogs

使用docker镜像技术在云服务器上以wordpress应用组容器 + mysql容器搭建个人博客网站的小问题记录。

我现在能清晰地想起来的就只有：

  1. 使用容器带来的巨大便利，扒一个现成镜像然后设置一下文件挂载、端口映射、网络模式什么的之后就可以直接跑起来一个容器，wordpress + Apache 和 mysql 之类的应用组和应用就可以直接使用了。太🐂了
  2. 在为 wordpress 配置数据库时遇到一点小问题：
    • 我使用了云服务器上本地数据库 + 本地 wordpress 应用组的模式。没有使用远程服务器。在 wordpress 初始化配置界面要求填写“数据库主机”，这时候填写 "localhost"、"127.0.0.1" 都是无效的。
    • 解决办法是执行 docker inspect <mysql-container-name> | grep IPAddress 获取本地给 mysql 容器分配的 ipv4 地址（或者叫伪网络地址合适些），然后填写地址和端口即可。

非常好，非常详细的教程：Docker安装wordpress并配置数据库
