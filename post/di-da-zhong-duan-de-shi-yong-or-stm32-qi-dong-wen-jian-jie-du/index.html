滴答中断的使用

简要说明

滴答计时器 SysTick 是 STM32 系列处理器的一种片上外设，内嵌在 NVIC 中，其时钟信号来自于系统时钟，具体可查看对应处理器的时钟树。

使用 SysTick 的重点无非是：初始化时设置时钟分频以控制中断频率、中断处理函数的书写。在裸机上寄存器编程时还要注意计时器的打开和关闭。然后就没了。

代码与分析
基于 STM32F103VE 的 SysTick 初始化
// STM32F103VE 主频 72MHZ
uint32_t SystemCoreClock = SYSCLK_FREQ_72MHz;

/**
  * @brief  初始化系统滴答定时器 SysTick
  * @param  无
  * @retval 无
  */
void SysTick_Init(void)
{
    /*
	   SystemFrequency / 1000    1ms中断一次
	   SystemFrequency / 100000	 10us中断一次
	   SystemFrequency / 1000000 1us中断一次
	*/
    
	//	if (SysTick_Config(SystemFrequency / 100000))	// ST3.0.0库版本
    
    // 设置为 1ms 触发一次滴答中断
	if (SysTick_Config(SystemCoreClock/1000))	// ST3.5.0库版本 SystemCoreClock / 10 不能超过 16777216 = 2^24
	{ 
		/* Capture error */ 
		while (1);
	}
	// 关闭滴答定时器  
	SysTick->CTRL &= ~ SysTick_CTRL_ENABLE_Msk;
	  
	// 使能滴答定时器，在此处直接开启或在主函数中初始化完成后开启均可
	//	SysTick->CTRL |=  SysTick_CTRL_ENABLE_Msk;
}
滴答中断的处理函数

file name: stm32f10x_it.c

unsigned int Task_Delay[NumOfTask]={0};

#define TASK_DELAY_0    1000    //任务0延时 1000*1 毫秒后执行
#define TASK_DELAY_1    500     //任务1延时 500*1 毫秒后执行
// 还可以写更多，不过要考虑任务过多对 CPU 资源的占用

/**
  * @brief  滴答中断处理函数，时延处理多任务
  * @param  None
  * @retval None
  */
void SysTick_Handler(void)
{
	int i;
    for(i=0; i < NumOfTask; i++)
    {
        Task_Delay[i] ++;                   //任务计时，时间到后执行
    }
  
  	// 处理任务 0
  	if(Task_Delay[0] >= TASK_DELAY_0)     //判断是否执行任务0
  	{
    	Task_Delay[0] = 0;                  //置0重新计时
      
    	// 任务体 0
  	}
  
  	// 处理任务 1
  	if(Task_Delay[1] >= TASK_DELAY_1)     //判断是否执行任务1
  	{
    	Task_Delay[1] = 0;                  //置0重新计时
      
    	// 任务体 1
  	}
  	// 处理任务 2
}

STM32 启动文件解读

简要说明

无论使用裸机还是嵌入式操作系统，都需要启动文件。启动文件通常由和处理器型号匹配的汇编语言书写，如野火-指南者开发板，使用 STM32F103VE 处理器，内置 ARM 的 Cortex-M3 内核，其启动文件自然就用 ARM 的汇编指令书写。ARM V 几我没查（

我主要看了 STM32F10x 系列使用标准库和使用 HAL 库时的启动文件，基本内容大致相同：设置系统堆栈、设置中断向量表、设置代码段、（设置数据段）、设置用户堆栈。下面以使用 HAL 库时 ST 官方提供的启动文件 "startup_stm32f103xe.s" 作为样例进行代码分析

代码分析

__initial_sp

; Heap Configuration
; Heap Size (in Bytes) <0x0-0xFFFFFFFF:8>

__heap_base
Heap_Mem        SPACE   Heap_Size
__heap_limit

                PRESERVE8
                THUMB

两段代码相似度很高，这里只解释配置堆区的部分

  • EQU 命令定义了表示堆区大小的变量，此处堆区大小为 0
  • AREA 定义一个代码段或数据段或一块不指明用途的空间（如本例，空间名为 HEAP，没有说明是代码还是数据），以后续参数说明其特性
    • NOINIT 不初始化
    • READWRITE 可读可写
    • ALIGN=3 指示当前位置按 23=82^3 = 823=8 字节对齐
  • __heap_base 声明一个标签，表示堆内存的起始地址
  • SPACE 命令分配一块大小为 Heap_Size 的空间，接续上一条声明标签的指令，使堆内存从起始地址处扩展 Heap_Size
  • __heap_limit 声明一个标签，表示堆内存的结束地址

堆栈配置至此结束，下面两条是传递给汇编器的指令（应该可以看作编译参数吧🤡）

  • PRESERVE8 向汇编器申明，本文件对内存空间的操作始终保持 8 字节对齐，使汇编器可以标识文件的 PRES8 属性，以保证链接时不会报错

    • PRESERVE8 作为伪指令不会对内存空间作任何修改
    • 如此重视对齐问题是因为要遵循 ARM 的 [AAPCS 栈使用规约](Introduction to the Armv8-M Architecture and its Programmers Model User Guide)

    6.2.1.1 Universal stack constraints
    At all times the following basic constraints must hold:

    • Stack-limit ≤ SP ≤ stack-base. The stack pointer must lie within the extent of the stack
    • SP mod 4 = 0. The stack must at all times be aligned to a word boundary.
    • A process may only store data in the closed interval of the entire stack delimited by [SP, stack base - 1] (where SP is the value of register r13).

    6.2.1.2 Stack constraints at a public interface
    The stack must also conform to the following constraint at a public interface:

    • SP mod 8 = 0. The stack must be double-word aligned

    简单解释就是：一般的栈空间必须保证 4 字节对齐，在外部函数的调用接口处必须保证 8 字节对齐。
    如果写成 PRESERVE8 {FALSE}，部分编译告警信息如下：

    linking...

    编译器告警提示，在启动文件中规定了 PRESERVE8 {FALSE} 后就不应该再调用要求 8 字节对齐的函数，此处主要是一些中断处理函数。

    注意，在下文的中断向量表中弱定义了非常多中断处理函数，但此处只会对在 C 语言文件中有定义的那部分函数告警，很好地回应了在外部函数地调用接口处调用栈必须保持 8 字节对齐的要求。

看着吓人，实际上结构很清晰，相当于声明了一些默认的中断服务函数，并向中断向量表中填充函数地址。这里只说明几条新冒出来的指令

  • AREA RESET, DATA, READONLY，申请名为 RESET 的空间，作为 DATA 段（数据段），访问权限为只读
  • EXPORT 声明一个全局的标志，使该标志可以被其他文件引用
  • DCD 申请一个 Byte 的内存空间，并用其后的表达式初始化这片内存

在 ARM 汇编文件中，占据一片连续内存空间的变量，变量中只是显式地存储连续空间的首地址，这一点和 C 语言相同

在本例中，由 DCD 命令申请 1 Byte 内存空间，其地址存储到中断向量表变量 __VECTORS，空间中存储的初始化值是栈的结束地址 __initial_sp

  • AREA 指令申请数据段后，后续代码就要填充数据，申请代码段后，后续就要填充代码
  • PROC 指令标明一段子程序的开始，ENDP 标明一段子程序的结束，二者必须成对出现
  • ALIGN 参数缺省，默认为 4 字节对齐

这一段相当于定义了一些默认的中断服务函数，全部用 [WEAK] 标识为弱定义，表示可在其他文件中重写，并以 EXPORT 申明为可引用。上文拿中断函数地址填充中断向量表也依赖于此处的函数定义。

设置用户堆栈
                 IF      :DEF:__MICROLIB
                
                 EXPORT  __initial_sp
                 EXPORT  __heap_base
                 EXPORT  __heap_limit
    
                 ELSE
    
                 IMPORT  __use_two_region_memory
                 EXPORT  __user_initial_stackheap
                 
__user_initial_stackheap

                 LDR     R0, =  Heap_Mem
                 LDR     R1, = (Stack_Mem + Stack_Size)
                 LDR     R2, = (Heap_Mem +  Heap_Size)
                 LDR     R3, = Stack_Mem
                 BX      LR

                 ALIGN
                 ENDIF
                 END
  • 若定义了宏 __MICROLIB，则使用 Microlib 标准库编译 C 文件，并将三个标签 __initial_sp、__heap_base、__heap_limit 声明为外部可引用。后续会由 C 库函数 __main 自动完成用户栈初始化
  • 若没有定义 __MICROLIB，则引用外部程序 __use_two_region_memory，即使用双段存储模式，并声明全局可引用程序__user_initial_stackheap，接下来定义此子程序，定义也没有什么特殊的，就是保存堆栈地址和大小到寄存器 R0~R3，最后设置返回地址到 LR (R14)
