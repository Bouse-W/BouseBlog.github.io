OOP 和 OPP

C 语言中使用结构体封装数据就相当于 OOP 中封装类属性。虽然可以通过封装函数指针来模拟 OOP 中的类方法，但由于缺少一个类似 this 的概念，无法做到对结构体变量自身状态的封装和操作。

Strictly, when postfixed to a variable, the ++ operator means increment the variable, but return the variable's value before it was incremented. So the C statements int c = 1; int d = c++ ; will result in variable d having a value of 1 and variable c having a value of 2. This does not quite express the idea that C++ is an increment on C. ——《C++ 编程入门手册》

OPP 的两大原则：结构化编程、自顶向下划分问题。OOP 与之有很大不同

与强调算法的过程性编程不同的是，OOP 强调的是数据......其理念是设计与问题的本质特性相对应的数据格式。——《C++ Primer Plus》中文第六版

OOP 程序设计方法手心设计类......从低级组织（如类）到高级组织（如程序）的处理过程叫自下向上（bottom-up）的编程。——《C++ Primer Plus》中文第六版

C++ 的编译器

不同的编译选项（优化等级、链接参数等）可以用于生成不同版本的可执行文件（稳定版、测试版等）。

一些编译器甚至支持对运行时内存的格式和大小的设置，以及对函数传参顺序的指定。（如 MSBuild），只要设置好编译参数就行。编译器真的很厉害。

编译器有时也很笨，请见下示代码

int k = 1 / 0;

编译告警 warning: division by zero [-Wdiv-by-zero]。

int i  = 0;
int j = 1 / i;

编译正常通过。使用变量 j 时报告 浮点数 exception（核心已转储） 错误。

C/C++ 程序的返回值

The rule in C++ is that, if the function declares that it returns a value, then it must return a value. However, there is a single exception to this rule: if the main function does not return a value, then a value of 0 will be assumed.——《C++ 编程入门手册》

c/c++程序都必然的有 main 函数，其生成的可执行文件则必然地有执行返回值。在命令行窗口直接运行可执行文件后，会改变当前窗口的某个环境变量来存储返回值，Linux 和 Windows 中的这个环境变量各不相同。
